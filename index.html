毎日の使い方

インストールが終わったら、日常的に打つコマンドは実質 2 つだけ。Terminal を開いて、hermes と打つ。それで完結します。

◇ ポイント

ComfyUI のタブは「裏方」、Hermes のタブは「会話用」。ComfyUI のタブはログが流れ続けるが触らない（閉じると死ぬ）。Hermes のタブで自然言語で指示する。ブラウザは Web UI で直接ノード触りたい時だけ使う。

朝、PC を起動してから使い始めるまで

01

Terminal を開いて Ubuntu に入る

/ ほぼ何もしなくていい

スタートメニューから 「Terminal」 を起動。Section 01 で 既定のプロファイルを Ubuntu にしておけば、開いた時点で WSL Ubuntu の中に入っている状態になります。

もし PowerShell タブが開いた場合は wsl と打つだけ:

wsl

プロンプトがこの形なら Ubuntu の中:

username@HOSTNAME:~$
末尾が ~$ なら OK。
02

hermes と打つ

/ それだけ
hermes

対話モードが立ち上がって、自然言語で指示できる状態になります。例:

> ComfyUI で「sunset over mountains」を 4 枚生成して
> 生成した画像のファイル名を一覧で見せて
> 出力ファイルを Discord に送って
終わるときは Ctrl + D または /quit で抜ける。
◇ ComfyUI は裏で動いている

Section 06 で systemd 自動起動を設定していれば、PC 起動と同時に ComfyUI もバックグラウンドで立ち上がっています。Hermes が画像生成を頼まれた時に勝手に使ってくれるので、ユーザーが直接気にする必要はありません。

ノードグラフを手で組みたい時だけブラウザで http://localhost:8188 を開く。これは「毎日やる」ことではなく、ワークフロー設計したい時の選択肢。

よくある「あれ？」と対処

ブラウザで localhost:8188 が開けない
ComfyUI が動いていない。systemctl --user restart comfyui で再起動するか、systemd 未設定なら cd ~/comfy && .venv/bin/python main.py --listen 127.0.0.1 --port 8188 --disable-smart-memory を別タブで実行。
Hermes と打っても "command not found"
WSL に入れているか確認。プロンプトが username@HOSTNAME:~$ の形なら OK。PS C:\...> なら PowerShell。wsl を実行して Ubuntu に切り替える。
Ctrl + V で貼り付けが効かない
Ctrl + Shift + V または右クリックを使う。WSL 内の Ctrl + V は別のキーバインド。
wsl と打ったら違うシェルに入った
Docker Desktop 等が WSL に居座っている。PowerShell で wsl --set-default Ubuntu して、再度 wsl。
ComfyUI のサンプリング 50% で死ぬ
ComfyUI 起動コマンドに --disable-smart-memory が付いているか確認。systemd ユニットファイルに記述済みなら自動で付くので、systemd で起動していれば問題なし。
PC をシャットダウンして翌日もまた同じ手順？
そう。systemd 自動起動を設定していれば Step 1 と 3 だけで OK。設定していなければ Step 2 も必要。

終了するとき

気にしなくて OK。PC をシャットダウンすれば全部止まります。明示的に止めたいなら:

  • Hermes 対話モード: Ctrl + D または /quit
  • ComfyUI: タブを閉じる（または systemctl --user stop comfyui）
  • WSL ごと停止: PowerShell で wsl --shutdown

仮想化チェック — タスクマネージャーが最速

  1. Ctrl + Shift + Esc でタスクマネージャー起動
  2. パフォーマンス タブ → CPU を選択
  3. 右下に 「仮想化: 有効」 と表示されていれば OK

無効になっていた場合のみ BIOS/UEFI で Intel VT-x または AMD-V を有効化。

Windows Terminal を入れる — ほぼ必須

🔥 最重要

旧 PowerShell コンソールはクリップボード操作が非常に不便で、コマンドの貼り付けが効かず手打ち地獄になる。本ガイドを始める前に必ず Windows Terminal を入れること。

  1. スタートメニュー → Microsoft Store
  2. 「Windows Terminal」で検索 → 入手
  3. 完了後、スタートメニューから 「Terminal」 を起動

覚えておくショートカット:

  • Ctrl + Shift + V または右クリック … 貼り付け
  • Ctrl + Shift + C … コピー
  • Ctrl + Shift + T … 新規タブ
  • Ctrl + , … 設定

設定で 既定のプロファイルを Ubuntu に しておくと、毎回 Terminal を開けば自動的に Ubuntu タブが開きます。

— SECTION 02 / SETUP

WSL2 + Ubuntu

インストール

Windows Terminal（または PowerShell）を 管理者権限で 開いて:

wsl --install

これ一行で WSL 機能有効化 + Virtual Machine Platform 有効化 + WSL2 カーネル DL + Ubuntu インストールが入る。完了後、PC を再起動。

🔥 時間と進捗表示の罠

wsl --install は 15 分〜1 時間以上。進捗表示がほとんど出ない時間帯があるので「フリーズした？」と思って中断したくなるが、ほぼ確実にバックグラウンドで動いている。絶対に Ctrl+C で止めない、ウィンドウを閉じない、PC をスリープさせない。

進捗が見える分割インストール（推奨）

「黙ったまま 1 時間」が不安なら、2 ステップに分けるとフィードバックが明確:

# Step 1: WSL の枠組みだけ入れる（数分）
wsl --install --no-distribution

# 再起動

# Step 2: Ubuntu を別途インストール
wsl --install -d Ubuntu-24.04

Ubuntu の初回セットアップ

再起動後、スタートメニューから「Ubuntu」起動。プロビジョニングで 1〜3 分黙ることがある。

ユーザー名とパスワードを聞かれる:

Create a default Unix user account: <あなたのユーザー名>
New password:
Retype new password:
⚠ 落とし穴
  • パスワード入力中はカーソルが動かない。打てているので最後まで入力して Enter
  • 2 回目で違う文字を打つとエラー長文が出るが、Try again? [y/N] で y で再挑戦できる
  • シェル起動直後は /mnt/c/Users/...$ にいる。cd ~ でホームへ移動

WSL のバージョン確認

wsl --list --verbose

期待:

  NAME      STATE           VERSION
* Ubuntu    Running         2
⚠ Docker Desktop 同居問題

Docker Desktop 等が入っていると、* 印が docker-desktop に付き、wsl で違うシェルに迷い込む。対処:

wsl --set-default Ubuntu

初期更新

sudo apt update && sudo apt upgrade -y

WSL 設定ファイル — 推奨

Windows の C:\Users\<ユーザー名>\.wslconfig:

[wsl2]
memory=16GB
processors=8
swap=8GB
localhostForwarding=true

書き換えたら wsl --shutdown で反映。

NVIDIA GPU
を WSL から使う

🔥 絶対原則

Windows 側にだけ NVIDIA Driver を入れる。WSL 側に入れない。WSL 内で apt install nvidia-driver-* をやると壊れる。

Driver インストール

  1. NVIDIA Driver ページ
  2. 自分の GPU を選択 → Studio Driver（Game Ready より安定）
  3. DL → インストール → 再起動

動作確認

nvidia-smi

GPU 名と VRAM が表示されれば成功。CUDA Toolkit は入れる必要なし（PyTorch がバンドル）。

— SECTION 06 / SETUP

ComfyUI
セットアップ

Hermes に頼んで自動セットアップ

hermes 対話モードで:

ComfyUIをローカルにセットアップして。GPUは<あなたの GPU 名>で、VRAMは<容量>です。
インストール先は ~/comfy で、APIモード（ポート8188）で起動するようにしてください。

Hermes が全部やってくれる。30 分〜1 時間かかる（PyTorch 約 2GB DL がボトルネック）。

🔥「Dangerous Command」警告への対応

途中で出る:

⚠ Dangerous Command
  pip install --user pipx && pipx ensurepath

  1. Allow once
> 2. Allow for this session
  3. Deny

「2. Allow for this session」。pipx は PyPA 公式の正規ツール、誤検知です。

動作確認

セットアップ完了したら、Windows ブラウザで:

http://localhost:8188

ComfyUI のノードエディタが表示されれば成功。

常駐化（強く推奨）— systemd 自動起動

毎回手動で起動しなくて済むように、systemd ユーザーサービスを登録:

mkdir -p ~/.config/systemd/user

cat > ~/.config/systemd/user/comfyui.service <<EOF
[Unit]
Description=ComfyUI Server
After=network.target

[Service]
Type=simple
WorkingDirectory=$HOME/comfy
ExecStart=$HOME/comfy/.venv/bin/python main.py --listen 127.0.0.1 --port 8188 --disable-smart-memory
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
EOF

systemctl --user daemon-reload
systemctl --user enable --now comfyui
systemctl --user status comfyui

これで PC 起動時に自動で ComfyUI が立ち上がるようになる。毎日の使い方 Step 2 が省略できる。

HuggingFace の Xet 問題

⚠ ハッシュ名で保存される

HuggingFace の大容量ファイルを aria2c で普通に DL するとハッシュ名で保存されることがある:

77d11a095c65fa57e869056f95835b2b2b7d6fdd40720f581897db33daca9072

対策: -o でファイル名指定する、または huggingface-cli を使う。

# 推奨: -o でファイル名指定
aria2c -c -x 16 -s 16 \
  -o anima-preview3-base.safetensors \
  https://huggingface.co/.../anima-preview3-base.safetensors

# または huggingface-cli
~/comfy/.venv/bin/pip install huggingface_hub
~/comfy/.venv/bin/huggingface-cli download <repo> <path> --local-dir <dir> --local-dir-use-symlinks False

Anima（DiT 系）の特殊性

Anima は SDXL と違って3 ファイル別々に配置:

ファイル配置先サイズ
anima-preview3-base.safetensorsdiffusion_models/4.2 GB
qwen_3_06b_base.safetensorstext_encoders/1.2 GB
qwen_image_vae.safetensorsvae/254 MB
◇ モデル追加後は ComfyUI を再起動

ComfyUI はモデル一覧を起動時に 1 度だけスキャンする。新ファイルを追加したら必ず再起動:

systemctl --user restart comfyui

Zone.Identifier ファイル — 定期掃除

Windows からコピーすると残骸ファイルが付くことがある:

find ~/comfy/models/ -name "*Zone.Identifier*" -delete
— SECTION 08 / REFERENCE

運用パターン
3 つのルートを使い分け

ルート A — Web UI を直接使う

ブラウザで http://localhost:8188。ノードを手で組む。試行錯誤・新モデル動作確認に最適。

ルート B — Hermes に自然言語で頼む

hermes 対話モードで「これとこれをやって」と話す。バッチ自動化・複数ツール連鎖に強い。コンテンツポリシー対象は弾かれる。

ルート C — Python ランナーで API 直接

ComfyUI のワークフロー JSON を Python から API に投げる。Hermes が拒否する内容や、完全自動化に。

役割分担の最適解

ComfyUI Web UI でワークフロー設計、Python でバッチ実行、Hermes でオーケストレーション・通知。Hermes は中身を見ない物流係として使うとコンテンツポリシーに引っかからない。

— SECTION 10 / ADVANCED

応用
Discord 連携

外出先のスマホから Discord 経由で Hermes に話しかけられるようにする。所要 15〜25 分。

Bot 作成

  1. Discord Developer Portal で New Application
  2. 左メニュー Bot → Privileged Gateway Intents で Message Content Intent を ON
  3. Reset Token で Token をコピー（再表示不可）

サーバーに招待

  1. 左メニュー OAuth2
  2. Scopes: bot, applications.commands
  3. Bot Permissions: Send Messages / Read Message History / Attach Files / Embed Links
  4. 生成 URL をブラウザで開く → サーバー選択 → 認証

ID 取得

Discord で 設定 → 詳細設定 → 開発者モード を ON。

  • チャンネル ID: チャンネルを右クリック → チャンネル ID をコピー
  • ユーザー ID: 自分のユーザー名を右クリック → ユーザー ID をコピー

Hermes に Gateway 追加

hermes gateway add discord

対話式で Bot Token、Allowed Channel IDs、Allowed User IDs を入力。

常駐化

cat > ~/.config/systemd/user/hermes-discord.service <<EOF
[Unit]
Description=Hermes Discord Gateway
After=network.target

[Service]
Type=simple
ExecStart=$HOME/.local/bin/hermes gateway start discord
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
EOF

systemctl --user daemon-reload
systemctl --user enable --now hermes-discord

落とし穴
TOP 10

このガイドを読みながら進めても踏みやすい順。

  1. wsl --install の進捗が止まって見えてキャンセル → 待つ。1 時間級になることがある
  2. クリップボードが効かなくて手打ち地獄 → Windows Terminal を入れる + Ctrl + Shift + V
  3. Docker Desktop 等が入ってて wsl で違うディストロに入る → wsl --set-default Ubuntu
  4. ComfyUI が Terminal タブを閉じると死ぬ → systemd で常駐化（Section 06 末尾）
  5. pip install が PEP 668 エラー → ~/comfy/.venv/bin/pip を使う
  6. Hermes の "Dangerous Command" 警告で止まる → pipx は誤検知。Allow for this session
  7. HuggingFace の DL がハッシュ名になる → aria2c -o <filename> または huggingface-cli
  8. モデル追加したのに ComfyUI が認識しない → ComfyUI を再起動
  9. Hermes が NSFW を拒否してパイプラインが止まる → 役割分担。Python ランナーで API 直叩き
  10. ComfyUI のサンプリング 50% で突然死 → --disable-smart-memory 起動オプション必須
